	.	 正解 標準キューから送られるメッセージはFIFO(First In, First Out)キューと異なり、順序を保証しません。ですが、厳密ではない FIFO機能が備わっているため、負荷が低い状況の場合はキューにデータが詰められた順番通りにメッセージが配信されます。標準キューの方がFIFOキューよりも優れていない様に感じますが、その分ほぼ無制限のトランザクションをサポートしています。順序がそれほど大事ではないが、大量のリクエストが送られることが予想される場合は標準キューを利用しましょう。 ※参考 Amazon SQS の特徴     
	.	現在、スマホ向けのゲームアプリケーションを開発しており、EC2インスタンスを利用して環境構築しています。システム要件として、ゲームユーザの行動データ解析処理を高速に行える構成が要求されており、最適なデータベース処理の設計を行う必要があります。 要件を満たすためのアーキテクチャを選択してください。  
		 ElastiCacheによるキャッシュ処理を実装する。 Auroraによるクエリ処理を実装する。 RDSでのマルチAZ構成を実装する。 EMRによる高速処理を実装する。
	.	 正解 ElastiCacheは、高スループットかつ低レイテンシーなインメモリデータストアからデータを取得して、大量のデータを扱うアプリケーションを構築したり、既存のアプリケーションのパフォーマンスを改善したりすることが可能です。 「ElastiCacheによるキャッシュ処理を実装する。」ElastiCacheは完全マネージド型の インメモリDBです。オープンソースであるRedis および Memcached といった普及しているオープンソース互換のインメモリデータストアを、シームレスにデプロイ、運用、スケールできます。高スループットかつ低レイテンシーなインメモリデータストアからデータを取得して、大量のデータを扱うアプリケーションを構築したり、既存のアプリケーションのパフォーマンスを改善したりすることが可能です。「ゲームユーザー行動データの解析処理を高速なパフォーマンス」を提供するにはElastiCacheが最適となります。 「Auroraによるクエリ処理を実装する。」Amazon Aurora は、クラウド向けに構築され、MySQL や PostgreSQL と互換性のあるリレーショナルデータベースです。商業用データベースと同等のパフォーマンスと可用性を、10 分の 1 のコストで実現します。RDSと比較して高性能なRDBではありますが、ゲームユーザー行動データにはElastiCacheなどのNoSQL型DBでの処理が向いています。 「RDSでのマルチAZ構成を実装する。」は不正解です。RDSでのマルチAZ構成は可用性を向上させる仕組みであるため、パフォーマンスの高性能化とは無関係です。 「EMRによる高速処理を実装する。」は不正解です。Amazon EMR は、オープンソースのフレームワークである Apache Spark と Hadoop を使用して、膨大な量のデータを迅速かつコスト効率よく処理して分析するサービスです。ゲームユーザー行動データの解析処理には利用できなくはないですが、データベースではないため、比較するとElastiCacheの方が適したユースケースとなり要件により合致しているため、不正解となります。     
	.	次のプログラム言語のうち、AWS SDKでサポートされていないプログラム言語は次のうちどれですか？  
		 JavaScript と Perl JavaScript と Node.js Perl Node.js と Go
	.	 正解 AWS SDKでは、Perlはサポートされていません。C++、Go、Java、JavaScript、.NET、Node.js、PHP、Python、Rubyはサポートされています。 ※参考 AWS での構築ツール     
	.	WebサイトをAWSにて構築している。当該WebサイトはALBで提供しており、Route 53 で名前解決されるようDNSレコードを登録している。Webサイトのメンテナンス時は利用者に対して予め用意した静的ページを見せたい。オーバーヘッドを最小限に抑えながらこの要件を満たす場合の最適なAWSサービスの構成を選択してください。  
		 S3 バケットに、メンテナンス用Webページを配置する。Route 53 のヘルスチェック機能を有効化し、Active-passive フェイルオーバ構成に変更する。コンテンツを提供する ALB のヘルスチェックに失敗した場合、S3バケットのメンテナンス用WebページのIPを解決し、ユーザを誘導する 配布用のメンテナンス用Webページを用意し、CloudFront のディストリビューションとして設定した ALB に配置する。ALB の DNSレコードは、Route 53 にエイリアスレコードとして設定する 既存のDNSレコードを遅延ベースのルーティングポリシーに変更する。Amazon S3 バケットにメンテナンス用Webページをホスティングし、遅延が最も小さいエンドポイントからレスポンスされるように設定する メンテナンス用のWebページを、EC2インスタンスに配置し、前段にALBの構成を取る。DNSレコードは、Route 53 の Active-active フェイルオーバで構成し、ALB のヘルスチェックが失敗した場合に、EC2インスタンスへのリクエストを送信する
	.	 正解 「S3 バケットに、メンテナンス用Webページを配置する。Route 53 のヘルスチェック機能を有効化し、Active-passive フェイルオーバ構成に変更する。コンテンツを提供する ALB のヘルスチェックに失敗した場合、S3バケットのメンテナンス用WebページのIPを解決し、ユーザを誘導する。」が正解です。Route 53 では、同じリソースの種類のレコードに対し、Active-active あるいは Active-passive でフェイルオーバ構成を組めます。Active-passive の構成とする場合、ルーティングポリシーを「フェイルオーバ」に設定し、「Active-active」の場合、それ以外のルーティングポリシーを選択します。 ※参考 DNS フェイルオーバーの設定
